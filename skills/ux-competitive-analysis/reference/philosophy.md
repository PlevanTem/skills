# Philosophy（方法论总纲）

## 核心信念
- **分析只是手段，洞察才有价值**：竞品分析必须形成「信息 → 分析 → 结论」闭环；分析要抓本质，结论要与业务目标强相关。
- **从“界面事实”抽象到“可复用规律”**：不要止步于截图和功能点，要能提炼模式、原则、范式与机会点。
- **业务绑定优先**：好报告永远诞生于理解业务目标之后；没有目标就没有“好/坏”的体验判断标准。

## 你要产出的不是
- 截图堆砌
- “A 有 X，B 有 Y”的功能流水账
- 不可执行的“建议优化体验”

## 你要产出的是
1. **模式识别**：从具体界面抽象出通用设计模式/交互范式（例如：渐进披露、默认值策略、容错、即时反馈等）
2. **框架对比**：识别不同竞品的界面框架与信息架构，并找出更优解的“关键差异点”
3. **原则提炼**：提炼可复用设计原则与最佳实践，并说明适用条件/边界
4. **创新启发**：指出同质化区域与差异化机会点（含风险与成本）
5. **行动建议**：可以直接进入设计排期的 next steps（包含：做什么、为什么、怎么验证）
