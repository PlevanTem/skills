# Operational Modes（作业模式）

## Mode A：Wide-Angle Scanning（广域扫描 / 广度）
### Trigger
用户要“列出有哪些竞品/找灵感/市场概览/有什么方案”。

### Goal
- 找到直接/间接竞品与替代方案
- 做分层与生态位归类
- 建立“后续深挖清单”（优先级与选择理由）

### Search Strategy：5D Radar（发散搜索维度）
1. **Mainstream 主流榜单/评测**
   - 例：Best [Domain] tools 2025 / Top [Domain] software
2. **Upstarts 新锐/增长产品**
   - 例：New [Domain] tool / Product Hunt [Domain]
3. **Open Source 开源替代**
   - 例：GitHub [Domain] alternatives / Open source [Domain]
4. **Visuals 设计与概念侧视觉资产**
   - 例：Dribbble [Domain] UI / Behance [Domain] concept
5. **Ecosystem 生态集成/版图**
   - 例：[Domain] landscape map / [Domain] integrations

### Deliverables（A 模式产出）
- 竞品分层列表（直接/间接/替代/生态）
- 选择 3~6 个进入深挖的候选，并说明理由（用户匹配度、证据充分度、差异化潜力）

---

## Mode B：Focusing Deep-Dive（深度聚焦 / 深挖）
### Trigger
用户要对比“某个功能/某条任务流/某个转化点”，或要给“改进方案”。

### Goal
- 找到具体交互细节、关键流程、信息架构与支撑点（策略/合规/定价/权限/模型能力等）
- 产出差距分析与可落地的建议

### Search Strategy：Dive（定向搜索）
1. **Feature Specific 功能/流程定向**
   - 例：[Product] [Feature] UI flow / documentation
2. **Pricing & Plans 定价与版本差异**
   - 例：[Product] pricing / feature comparison
3. **User Content 用户内容（摩擦与吐槽）**
   - 例：YouTube review / Reddit complaints / forum feedback
4. **Direct Comparison 直接对比**
   - 例：[A] vs [B] [Feature]

### Deliverables（B 模式产出）
- 任务流级别拆解（每步输入/输出/反馈/容错/成本）
- 体验五要素/五层体验定位问题与亮点
- 差距分析：他们有什么我们没有；他们哪里做得不对
- 行动建议：我们该借鉴什么模式、要避免什么坑、怎么验证

---

## Mode A+B（组合：先广后深）
### Trigger
用户既要“找产品”，又要“深度对比/给方案”。

### Goal
先把地图画对，再把关键战场挖深。
