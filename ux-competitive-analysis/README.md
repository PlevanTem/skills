# 竞品分析 Skill（体验设计向）

专为体验设计师打造的竞品分析技能包，遵循「信息 → 分析 → 结论」闭环：
- 5D 雷达全网发现竞品（主流/新锐/开源/视觉/生态）
- 体验五要素（五层体验）+ 任务流拆解
- 强制截图证据 + 来源引用
- 输出核心洞察、可复用设计原则与可落地行动建议（拒绝流水账功能对比）

> 默认中文输出；批评必须解释原因与影响，并给出可执行下一步。

---

## 包含内容
- `SKILL.md`：技能入口（什么时候用、输出规则、执行导航、纠偏）
- `reference/`：方法论与框架（philosophy / intake / modes / frameworks）
- `checklists/`：证据采集清单（截图数量、来源优先级、重试策略）
- `templates/`：报告与引用模板（最终落到 `~/task_name/report.md`）

---

## 安装（最简单方式）
1. 下载或克隆本目录：`ux-competitive-analysis/`
2. 将整个目录放到你管理 Skills 的位置（按你的使用环境约定即可），并确保 `SKILL.md` 可被加载/引用。

> 这是“纯文本 Skill 包”：不依赖代码运行环境。

---

## 使用方法（推荐流程）
### 1) 启动任务（以设计师口吻描述需求）
在对话里直接提出竞品分析需求即可，例如：
- “我是体验设计师，做 IDE 智能编码助手插件。请先盘点主流竞品，再深挖其中 3 个在‘代码生成 + 可控性’上的关键任务流，输出可落地建议，并附截图与引用。”

如果你的需求不够明确，本 Skill 会先进行**需求补齐**（业务目标/用户/场景/指标/约束），然后再进入搜索与拆解。

### 2) 选择模式
- 只要“有哪些竞品/市场概览”：使用广域扫描（Mode A）
- 只要“某功能/某流程深挖对比”：使用深度聚焦（Mode B）
- 既要盘点又要对比：组合模式（Mode A+B）

模式细节见：`reference/operational-modes.md`

### 3) 产出物
- 竞品分析报告：`~/task_name/report.md`
- 竞品截图资产：`~/task_name/images/`（每个竞品 ≥ 3 张有效 UI 截图）

报告结构见：`templates/report-template.md`  
引用规范见：`templates/citation-style.md`

---

## 如何扩展（不改变方法论的前提下）
你可以只做“增量素材”，不改框架本身：
- 在 `reference/` 新增某领域的“默认关键词库/竞品清单”（例如 AI IDE、B2B SaaS、跨境电商等）
- 在 `templates/` 增加你团队常用的“评审维度/指标口径”
- 在 `checklists/` 增加你团队的“合规/可访问性/国际化”检查项
